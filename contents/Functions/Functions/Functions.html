Functions: the big picture

A function is a reusable section of code written to perform a specific task in a program. Functions provide better modularity for your application and a high degree of code reusing.

Python gives you many built-in function like print(), but you can also create your own functions. The functions are called user-defined functions

The values that the user gives to the functions when he call it to perform its job are called arguments or parameters depending of the context. The argument is the piece of code or values that actually is given to the function when it's called in execution time, while the parameter is the name you use when you write the functions code, to identify the future given values

You can think of parameters as nicknames the functions definition gives to arguments.

Function syntax

Defining a function

Functions are defined using the keyword def, and consist on three parts:

Header

The block begin with the keyword del followed by the function names, parentheses (within which are all parameters the function takes), and finally a colon (:).

Docstring

A triple-quoted, multi-line comment that briefly explains what the functions does.

Body

Is the code block that describes the procedures the functions carries out. The statement return exits a functions, optionally passing back an expression to the caller. A return statement with no arguments is the same as return None

def function_name (<parameters>):
	<"""documentation string""">
	body return <expression>

Calling a functions

Defining a functions only gives it a name, specifies the parameters that are to be included in the function, and structures the blocks of code.

Once the basic structure of a function is finalized, you can execute it by calling it from another function or dircetly from any part of the program.

To call a function you only need to write its name followed by his arguments within parentheses. If the functions doesn't need any arguments, the parentheses will appear empty

#calling function square(n) on 9
square(9)

#calling function random_real
random_real()

Function arguments

A function can be call by using the following tyes of formal arguments:

  1. Required arguments

  2. Keyword arguments

  3. Default arguments

  4. Variable-length arguments

Required arguments

Required arguments are the arguments passed to a function in correct positional order. Here the number of arguments in the function call should match exactly with the function definition to avoid syntax errors.

Keyword arguments

Keyword arguments are related to the function calls. When keyword arguments are used in a function call, the caller identifies the arguments by the parameter name and not by the order.

This python capability allows you to skip arguments or place the out of order because the python interpreter is able to use the keywords provided to match the values with parameters, e.g.,

# Function definition is here
def printinfo( name, age ):
   	"""This prints a passed info into this function"""
   	print "Name: ", name;
   	print "Age ", age;
   	return;

# Now you can call printinfo function
printinfo( age=50, name="miki" );

Default arguments

A default argument is an argument that assumes a default value is a value is not provided in the function call for the argument. It's import to remember to avoid the use of keyword arguments, the arguments with default values has to be listed at the end of the parameters chain in the function definition. Otherwise you will have to use keyword arguments to ensure the interpreter can differentiate them.

The next examples show the discused topic:

#using default arguments without keywords
def print_info (name, age = 15):
	print str(name) + " is " + str(age) + " years old."
			
#calling the function
print_info("name_1")

#using default arguments with keywords
def print_info (name = "John Due", age):
	print str(name) + " is " + str(age) + " years old."

#calling the function
	print_info(age = 16)

Variable number fo arguments

It could be necessary to process a function for more arguments than you specified while defining the function. These arguments are called variable-length arguments. Because it will be imposible to define a name for each of the "unknown number" arguments, that why only one container name is given when defining the function, the name of this container is presceded by an asterisk (*). This tuple remains empty if no additional arguments are specified during the function call.

The general syntax for a function with non-keyword variable arguments is this:

def functionname(<formal_args,> *var_args_tuple ):
   	"""function_docstring"""
   	function_suite
   	return <expression>

Following is a simple example:

# Function definition is here
def printinfo( arg1, *vartuple ):
   	"This prints a variable passed arguments"
   	print "Output is: "
   	print arg1
   	for var in vartuple:
     	 	print var
   	return;
   			
# Now you can call printinfo function
printinfo( 10 );
printinfo( 70, 60, 50 );

When the above code is executed, it produces following result:

Output is:
10
Output is:
70
60
50

Anonymous functions

The lambda keyword can be used to create small anonymous functions. This functions are called anonymous because they are not declared in the standard manner by using the def keyword.

Some of the characteristics of the anonymous functions are:

  • Can take any number of arguments but return just one value in the form of a expression.

  • They cannot contain commands of multiple expressions

  • An anonymous function cannot be a direct call to print

  • Have their own local namespace and cannot access variables other than those in their parameter list and those in the global namespace

  • They are not the equivalent to inline statements in C or C++

The syntax of lambda functions contains only a single statemnt, which is as fallows:

lambda <arg1 <, arg2, ..., argn>>: expression

Following is the example to show how lambda form of functions works:

# Function definition is here
sum = lambda arg1, arg2: arg1 + arg2;

# Now you can call sum as a function
print "Value of total : ", sum( 10, 20 )
print "Value of total : ", sum( 20, 20 )

Built-in functions

There's some functions that python gives you without importing any module, this functions are called built-in functions. Here is the list of built-in functions of python:

max()

takes any number of arguments (separated by commas) and returns the largest one

min()

given a series of arguments (separated by commas), it returns the smallest one.

abs()

returns the absolute value of the number it takes as an argument

type()

returns the type of the data it receives as an argument

dir()

returns a sorted list of strings containing the names defined by a module. The list contains the names of all the modules, variables, and functions that are defined in a module

str()

converts the variable content into a string
